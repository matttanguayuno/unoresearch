Documentation is changing

Docs are not just for humans anymore. They now need to work for people, LLMs, and agents/tools too.

The short version

The biggest shift is that we should stop thinking about everything as just "docs". There are really a few different layers now:

📖

Traditional Docs

Help humans understand things

▾

Still the foundation. People need tutorials, how-to guides, reference pages, and explanation. This part has not gone away.

🤖

LLM-Ingestible Docs

Help models retrieve the right context

▾

Documentation structured so models can use it well. The key idea is not "AI magic" - it is mostly clean structure, good headings, self-contained sections, and markdown or plain-text outputs.

🔧

Tools

Let models take actions

▾

A tool is not just a page explaining a feature - it is something callable, with clear inputs, outputs, and expected actions. Tool documentation needs to be much more explicit than normal prose.

💬

Prompts

Guide how a model approaches a task

▾

Reusable guidance - useful for framing, templates, and repeatable instructions. But usually not the best long-term place to store important operational knowledge.

⚡

Skills

Package repeatable workflows or playbooks

▾

Probably the most interesting layer. A skill is a reusable playbook the model can load when needed - a great fit for repeated internal workflows, conventions, and step-by-step processes.

Why this matters

Old model

Write docs pages for humans

→
New model

Write once, then publish in a few ways depending on the job

So the same source material might become:

  • A normal docs page for people
  • A markdown or LLM-friendly version for retrieval
  • A tool schema for action
  • A skill for repeatable workflows

The real shift is not just "make docs AI-friendly".

It is: separate context, actions, and workflows clearly.

When those are separated:

📄 Docs are easier to read
🔍 Retrieval gets cleaner
🛡️ Tools become safer and more reliable
🔁 Repeated processes are easier to automate

Recommendations

1

Keep strong human docs

Good human docs still matter - and they also improve machine ingestion downstream.

2

Add an LLM-ingestible layer

Think clean markdown, strong headings, self-contained sections, and possibly an llms.txt-style entry point.

3

Treat actions as tools, not prose

If something can be done by an agent, describe it like a callable capability, not just paragraphs.

4

Use prompts lightly

They are useful, but should not become the dumping ground for every important process.

5

Turn repeated workflows into skills

That is likely where a lot of the practical value is.

Bottom line

Docs are turning into a system, not just a website.

Human Docs LLM Context Tools Prompts Skills

Each has a different job. The teams that separate those jobs clearly will be in a much better place for search, copilots, agents, and internal AI workflows.
